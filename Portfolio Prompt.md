# Award-Quality Portfolio Website Specification

## Vision

Build a **premium, award-quality personal portfolio website** that feels
closer to a world-class product website than a traditional developer
portfolio.

The experience should communicate craftsmanship, engineering excellence,
and attention to detail before the visitor reads a single paragraph.

### Inspiration

-   Linear
-   Vercel
-   Apple
-   Stripe
-   Raycast
-   n8n
-   Otsuka Air (scrolling experience)
-   Land-book featured websites

**Do not copy any website.** Take inspiration from their quality,
motion, typography, and storytelling.

------------------------------------------------------------------------

# Design Language

-   Pure **Black**, **White**, and **Gray** only.
-   No gradients.
-   No glassmorphism.
-   No neon.
-   No colorful UI.
-   The only colors allowed are those inside project media (Flutter
    apps, videos, etc.).

Use typography, spacing, and motion instead of color to create
hierarchy.

Typography should be: - Large - Bold - Elegant - Minimal - Spacious

The website should feel premium, cinematic, and understated.

------------------------------------------------------------------------

# Build Lab (Unique Feature)

This should become the signature section of the portfolio.

Instead of simply displaying projects, visitors can **launch and use
them**.

Layout:

    BUILD LAB

    DailyTask        [ Launch ]
    Vault            [ Launch ]
    Relay            [ Watch Demo ]
    n8n Companion    [ Launch ]

When Launch is clicked:

-   iPhone expands smoothly
-   surrounding UI fades
-   Flutter app becomes fully interactive
-   close returns to portfolio with elegant animation

This should feel like opening a real mobile application.

# Motion Principles

Every section should have purposeful motion.

Examples: - reveal animations - fade transitions - scale on scroll -
sticky storytelling - page transitions - smooth transforms

Animations should support the story, never distract.

------------------------------------------------------------------------

# Performance

-   Strong typing
-   Clean architecture
-   Accessible
-   Responsive

# Final Goal

This should not feel like a developer portfolio.

It should feel like:

**Apple × Linear × Vercel × n8n**

Visitors should spend several minutes exploring the site, interacting
with the Build Lab, trying the Flutter applications, and watching
automation workflows execute.

The final product should be memorable enough that recruiters and clients
remember it long after leaving the website.
